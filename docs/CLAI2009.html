Sintaxis de fórmulas proposicionales

type SimboloProposicional = String Source #

Los símbolos proposicionales se representan por cadenas.

data FProp Source #

FProp es el tipo de las fórmulas proposicionales definidas por

  • T y F son fórmulas
  • Si A es una fórmula, también lo es ¬A.
  • Si A y B son fórmulas, entonces (A ∧ B), (A ∨ B), (A → B) y (A ↔ B) también lo son.

Semántica de la lógica proposicional

type Interpretacion = [FProp] Source #

Las interpretaciones son listas de fórmulas atómicas. Las fórmulas de las interpretaciones se suponen verdaderas y las restantes fórmulas atómicas se suponen falsas.

significado :: FProp -> Interpretacion -> Bool Source #

(significado f i) es el significado de la fórmula f en la interprestación i. Por ejemplo, 

>>> significado ((p ∨ q) ∧ ((no q) ∨ r)) [r]
False
>>> significado ((p ∨ q) ∧ ((no q) ∨ r)) [p,r]
True

Modelos de una fórmula

simbolosPropForm :: FProp -> [FProp] Source #

(simbolosPropForm f) es el conjunto formado por todos los símbolos proposicionales que aparecen en f. Por ejemplo, 

>>> simbolosPropForm (p ∧ q → p)
[p,q]

interpretacionesForm :: FProp -> [Interpretacion] Source #

(interpretacionesForm f) es la lista de todas las interpretaciones de la fórmula f. Por ejemplo,

>>> interpretacionesForm (p ∧ q → p)
[[],[p],[q],[p,q]]

esModeloFormula :: Interpretacion -> FProp -> Bool Source #

(esModeloFormula i f) se verifica si la interpretación i es un modelo de la fórmula f. Por ejemplo, 

>>> esModeloFormula [r]   ((p ∨ q) ∧ ((no q) ∨ r))
False
>>> esModeloFormula [p,r] ((p ∨ q) ∧ ((no q) ∨ r))
True

modelosFormula :: FProp -> [Interpretacion] Source #

(modelosFormula f) es la lista de todas las interpretaciones de la fórmula f que son modelo de F. Por ejemplo,

>>> modelosFormula ((p ∨ q) ∧ ((no q) ∨ r))
[[p],[p,r],[q,r],[p,q,r]]

Fórmulas válidas, satisfacibles e insatisfacibles

esValida :: FProp -> Bool Source #

(esValida f) se verifica si la fórmula f es válida. Por ejemplo,

>>> esValida (p → p)
True
>>> esValida (p → q)
False
>>> esValida ((p → q) ∨ (q → p))
True

prop_esValida :: FProp -> Bool Source #

(prop_esValida f) se verifica si las siguiente condiciones son equivalentes:

  • f es válida
  • f es consecuencia del conjunto vacío.

+++ OK, passed 100 tests.

esInsatisfacible :: FProp -> Bool Source #

(esInsatisfacible f) se verifica si la fórmula f es insatisfacible. Por ejemplo,

>>> esInsatisfacible (p ∧ (no p))
True
>>> esInsatisfacible ((p → q) ∧ (q → r))
False

esSatisfacible :: FProp -> Bool Source #

(esSatisfacible f) se verifica si f es satisfacible. Por ejemplo,

>>> esSatisfacible (p ∧ (no p))
False
>>> esSatisfacible ((p → q) ∧ (q → r))
True

Interpretaciones de un conjunto de fórmulas

unionGeneral :: Eq a => [[a]] -> [a] Source #

(unionGeneral x) es la unión de los conjuntos de la lista de conjuntos x. Por ejemplo,

>>> unionGeneral []
[]
>>> unionGeneral [[1]]
[1]
>>> unionGeneral [[1],[1,2],[2,3]]
[1,2,3]

(simbolosPropConj s) es el conjunto de los símbolos proposiciones de s. Por ejemplo,

>>> simbolosPropConj [p ∧ q → r, p → r]

>>> interpretacionesConjunto [p → q, q → r]

Consecuencia lógica

esConsecuencia :: [FProp] -> FProp -> Bool Source #

(esConsecuencia s f) se verifica si f es consecuencia de s. Por ejemplo,

>>> esConsecuencia [p → q, q → r] (p → r)
True
>>> esConsecuencia [p] (p ∧ q)
False

prop_esConsecuencia :: [FProp] -> FProp -> Bool Source #

(prop_esConsecuencia s f) verifica que son equivalentes:

  • f es consecuencia de s
  • s ∪ {¬f} es inconsistente

+++ OK, passed 100 tests.
